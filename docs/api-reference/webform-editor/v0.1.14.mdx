# Intro

This is the webform editor package used in the Qodly Studio Editor

# Installation and Setup

To fix issues and develop new features this package and see the changes reflected directly on Qodly without friction we use the `yalc` package, please make sure that it's installed on your machine by running:

```bash
npm i -g yalc
```

## Step 1

Clone the webstudio project to your machine

```bash
git clone git@git-ps.wakanda.io:4d/web-studio/webstudio.git
```

## Step 2

Setup the project to work with the link to the local version of the project in the `craco.config.js` go to 

```javascript
const yalcDeps = { /*  */ }
```

and uncomment the line where is `'@ws-ui/webform-editor': { requiredVersion: '^0.0.1' },` is located

then run

```bash
yalc add @ws-ui/webform-editor && yalc link @ws-ui/webform-editor && npm install
```

## Step 3

Develop locally by running

```bash
npm start
```

### useEnhancedEditor

• **useEnhancedEditor**: 

**`Description`**

A Hook that provides methods and state information associated with the entire editor.

**`Param`**

A function that collects relevant state information from the editor state. The component will re-render when the values returned by this function changes.

**`Example`**

Collecting state information

<Tabs>
   <TabItem value="Example 1">

   ```tsx
   import { useEnhancedEditor } from "@ws-ui/webform-editor";

   const Example = () => {
     const { hoveredNodeId } = useEnhancedEditor((state) => ({
       hoveredNodeId: state.events.hovered
     }));

   return (
     <div>
 The ID of the node currently being hovered is: {hoveredNodeId}
     </div>
    )
   }
   ```

 </TabItem>
  <TabItem value="Example 2">

  ```tsx

The ID of the node currently being hovered is: {hoveredNodeId}
    </div>
  ```

### useLayout

**`Description`**

A Hook that provides methods and properties giving information about the webform airy mode

**`Example`**

Getting the AiryMode className

   ```tsx
   }
   ```
